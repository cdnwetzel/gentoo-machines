# Surface Pro 6 — Gentoo Install Execution Sequence
# Run from Fedora 43 Live USB (Ventoy)

## Pre-Requisites
- [ ] Secure Boot DISABLED in Surface UEFI (hold Volume Up + Power on boot)
- [ ] Booted Fedora 43 live USB via Ventoy
- [ ] WiFi connected (NetworkManager on live USB)
- [ ] Ventoy USB mounted at `/run/media/liveuser/VTOYEFI/`
- [ ] Git repo cloned at `/home/liveuser/gentoo-machines/`

## Step 1: Partition & Format NVMe

```bash
cd /run/media/liveuser/VTOYEFI/surfacepro6
sudo bash gentoo_install_part1.sh
```

**What it does:**
- Wipes `/dev/nvme0n1` (destroys Fedora + all data)
- Creates GPT: nvme0n1p1 (512M EFI), nvme0n1p2 (1G boot), nvme0n1p3 (~237G root)
- Formats: FAT32, ext4, ext4
- Mounts to `/mnt/gentoo`, `/mnt/gentoo/boot`, `/mnt/gentoo/boot/efi`

**Prompts:** Confirm disk wipe (type `YES`), confirm layout (type `y`)

## Step 2: Stage3 + Config Staging

```bash
sudo bash gentoo_install_part2.sh
```

**What it does:**
- Downloads stage3-amd64-desktop-openrc (~700MB)
- Verifies SHA512 checksum
- Extracts to `/mnt/gentoo`
- Copies all config files to `/mnt/gentoo/root/surface-pro-6-configs/`:
  - make.conf, world, package.use, kernel config script, base .config
  - LightDM config, openrc-services, restore scripts, touchpad config
- Copies shared portage files to `/mnt/gentoo/etc/portage/`
- Generates `/etc/fstab` with real UUIDs from blkid
- Mounts pseudo-filesystems (proc, sys, dev, run)
- Copies DNS config for chroot networking

**Prompts:** None (fully automated)

## Step 3: Enter Chroot

```bash
sudo chroot /mnt/gentoo /bin/bash
source /etc/profile
export PS1="(chroot) \[\033[0;31m\]\u@\h \[\033[0;36m\]\w \$ \[\033[0m\]"
```

## Step 4: One-Shot Install (all 13 phases)

```bash
bash /root/gentoo_install_part3_chroot.sh
```

**What it does (13 phases):**

| Phase | Action | Duration |
|-------|--------|----------|
| 1 | Bootstrap: emerge-webrsync, set profile, update @world | 10-20 min |
| 2 | Kernel: gentoo-sources + linux-firmware + build -j9 | 15-30 min |
| 3 | Bootloader: GRUB EFI install + config | 2 min |
| 4 | System: timezone, locale, hostname, root/user passwords | 2 min |
| 5 | Networking: wpa_supplicant + NetworkManager (CRITICAL) | 5 min |
| 6 | All packages: emerge @world (XFCE, PipeWire, browsers, etc.) | 1-2 hours |
| 7 | Portage: tmpfs dirs, ccache, package.env verification | 1 min |
| 8 | Services: rc-update for all OpenRC services | 1 min |
| 9 | LightDM: session=xfce, Xsession wrapper, display-manager | 1 min |
| 10 | Audio: verify PipeWire + WirePlumber installed | 1 min |
| 11 | Surface HW: zram-init config, firmware verification | 1 min |
| 12 | fstab: verify (already generated by part2) | 1 min |
| 13 | **Verification gate**: 15+ checks, FAIL count reported | 1 min |

**Prompts:**
- Root password (Phase 4)
- User `chris` password (Phase 4)
- Open menuconfig? y/N (Phase 2 — optional review)

**Total estimated time: 2-3 hours** (mostly waiting on emerges)

## Step 5: Exit & Reboot

Only if Phase 13 reports **ALL CHECKS PASSED**:

```bash
exit                                    # leave chroot
cd /
umount -l /mnt/gentoo/dev{/shm,/pts,}
umount -R /mnt/gentoo
reboot
```

Remove Ventoy USB when the screen goes dark.

## Step 6: Post-Boot Verification

After GRUB boots into Gentoo and LightDM shows the login screen:

```bash
# Connect WiFi
nmtui

# Verify display
xrandr                              # should show 2736x1824

# Verify audio
pactl info | grep "Server Name"    # should show PipeWire

# Verify zram swap
swapon --show                       # should show zram0, 4G, zstd

# Verify sensors
sensors                             # should show coretemp values

# Verify brightness control
xbacklight -get                     # should return a value (0-100)

# Verify GPU
lspci -k | grep -A2 VGA             # should show i915

# Verify kernel
uname -r                            # should show gentoo-sources version
```

## Step 7: Restore Desktop

```bash
cd ~
git clone https://github.com/cdnwetzel/gentoo-machines.git
cd gentoo-machines
bash shared/restore-desktop.sh       # XFCE keybindings, panels, displays
sudo bash shared/restore-system.sh   # elogind, ACPI lid, LightDM display setup
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| No WiFi after boot | `wpa_passphrase "SSID" "pass" > /tmp/wpa.conf && sudo wpa_supplicant -B -i wlp1s0 -c /tmp/wpa.conf && sudo dhcpcd wlp1s0` |
| Black screen at login | Check `/var/log/lightdm/lightdm.log`, verify `user-session=xfce` in `/etc/lightdm/lightdm.conf` |
| No sound | `pactl info`, check `emerge -pv pipewire \| grep sound-server`, run `gentoo-pipewire-launcher &` |
| No display/GPU wedged | Verify `DRM_I915=m` (not =y), check `dmesg \| grep i915` |
| zram not active | `cat /proc/swaps`, verify `CONFIG_ZRAM=y` + `CONFIG_CRYPTO_ZSTD=y` in kernel |
| GRUB not found | Re-chroot: mount partitions, `grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=Gentoo` |
